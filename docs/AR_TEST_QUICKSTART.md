## 🐛 常见问题

### Q1: 本地 HTTP 无法测试 AR

**原因**：WebXR 要求 HTTPS，本地 HTTP（<http://192.168.1.113:8080）不满足安全要求>

**解决**：
1. **使用 Vercel 在线版本**：<https://quest3-exploded.vercel.app>
2. **或配置本地 HTTPS**（复杂，推荐使用 Vercel）

**预期结果**：test-ar.html 会显示 HTTPS 检测失败，这是正常的。

---

### Q2: AR 按钮不显示

**可能原因**：
1. 桌面浏览器（不支持）
2. 浏览器版本太低
3. 未使用 HTTPS

**解决**：
1. 使用移动设备（Android/iOS）
2. 使用 Chrome（Android）或 Safari（iOS）
3. 访问 Vercel HTTPS 地址

---

### Q3: 点击 AR 按钮无反应

**排查**：
1. 检查浏览器控制台（远程调试）
2. 查看错误信息
3. 确保已授予相机权限

---

### Q4: 模型看不见

**解决**：
1. 在光线充足的环境
2. 扫描纹理丰富的平面
3. 移动手机速度放慢
4. 确保模型在视野内

---

## 🔍 远程调试

### Android Chrome

1. 手机开启**开发者选项**
   - 设置 → 关于手机 → 连续点击版本号
2. 连接 USB 到电脑
3. 电脑打开 Chrome
4. 访问：`chrome://inspect`
5. 选择设备 → 检查

### iOS Safari

1. 手机开启**开发者选项**
   - 设置 → Safari → 高级 → Web 检查器
2. 连接 USB 到 Mac
3. Mac 打开 Safari
4. 菜单：开发 → [设备名] → 页面
