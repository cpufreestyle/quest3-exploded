## 🔧 技术细节

### WebXR 检测流程

```javascript
1. 检查 'xr' in navigator
2. 调用 navigator.xr.isSessionSupported('immersive-ar')
3. 根据结果显示/隐藏 AR 按钮
```

### AR 会话流程

```javascript
1. 用户点击 AR 按钮
2. 请求 AR 会话
   - requiredFeatures: ['hit-test', 'local-floor']
   - optionalFeatures: ['dom-overlay', 'light-estimation']
3. 创建 AR 渲染器
4. 克隆 Quest 3 模型（缩放 10%）
5. 开始 AR 渲染循环
6. Hit-test 检测平面
7. 自动放置模型
8. 自动旋转 + 呼吸效果
```

### Hit-test 流程

```javascript
1. 用户移动手机扫描环境
2. frame.getHitTestResults() 检测平面
3. 获取 hit pose
4. 将模型定位到 hit point
5. 设置 arModelPlaced = true
```

### AR 渲染循环

```javascript
arRenderer.setAnimationLoop((timestamp, frame) => {
  1. 获取 viewer pose
  2. 更新相机投影矩阵
  3. Hit-test（如果未放置）
  4. 放置模型（如果检测到平面）
  5. 应用爆炸呼吸效果
  6. 自动旋转
  7. 渲染场景
});
```

## 📝 已知问题和解决方案

### 问题 1：找不到免费模型

**症状**：搜索后没有找到可用的 Quest 3 GLB 文件

**影响**：
- AR 中显示简化版
- 真实感不足

**解决**：
1. **方案 A**：手工在 Blender 中创建
2. **方案 B**：使用 AI 生成（如 Midjourney + 3D 化）
3. **方案 C**：从 iFixit 拆解图手动建模
4. **方案 D**：联系 Meta 开发者获取授权

**推荐**：方案 A（手工建模），质量可控

### 问题 2：AR 退出重新加载

**症状**：退出 AR 时页面重新加载

**影响**：
- 丢失当前状态
- 用户体验不佳

**临时方案**：
- 接受简化实现

**长期方案**：
- 保存状态到 localStorage
- 恢复 UI 和场景状态

### 问题 3：桌面无 AR

**症状**：桌面浏览器不支持 AR

**说明**：
- 这是预期的行为
- AR 需要移动设备传感器

**解决方案**：
- 仅在移动设备显示 AR 按钮
- 桌面用户可使用普通 3D 模式
