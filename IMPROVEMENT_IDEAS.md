# Quest 3 爆炸图 - 改进建议

## 🎯 核心定位
**交互式 VR 设备拆解教学工具**，目标用户：VR 新手、学生、开发者、3D 爱好者

---

## 📊 优先级分级

### 🔴 P0 - 核心缺失（必须做）

#### 1. **部件点击高亮**
**问题**：用户无法识别当前步骤拆的是哪个部件
**方案**：
- 步骤说明里标注部件名称
- 点击步骤描述中的部件名，3D 视图高亮该部件
- 高亮效果：发光 + 放大 + 其他部件变暗
- 添加"查找部件"按钮，自动旋转到该部件视角

```javascript
// 示例代码
function highlightPart(partName) {
  parts.forEach(p => {
    if (p.name === partName) {
      p.mesh.material.emissive = new THREE.Color(0x4a9eff);
      p.mesh.scale.setScalar(1.1);
    } else {
      p.mesh.material.opacity = 0.5;
    }
  });
}
```

#### 2. **工具清单**
**问题**：iFixit 每个步骤都列出所需工具，但我们没有
**方案**：每步显示：
- 🪛 **螺丝刀**：十字 #00
- 🔧 **撬棒**：塑料撬棒
- 📋 **步骤卡片**：详细图文说明

#### 3. **移动端适配**
**问题**：目前只在桌面端测试，移动端体验未知
**方案**：
- 触摸手势优化（双指缩放、旋转）
- 左侧面板改为底部抽屉
- 按钮更大更易点击（最小 44×44px）
- 横屏/竖屏自适应

---

### 🟠 P1 - 体验增强（强烈建议）

#### 4. **动画时间轴控制**
**问题**：只能看动画，不能精确控制
**方案**：添加进度条（时间轴）
- 播放/暂停按钮
- 可拖拽的时间轴
- 播放速度控制（0.5x / 1x / 2x）
- 当前步骤时间轴标记

```html
<div class="timeline">
  <button id="play-pause">⏸️</button>
  <input type="range" id="timeline-slider" min="0" max="100" />
  <select id="playback-speed">
    <option value="0.5">0.5x</option>
    <option value="1" selected>1x</option>
    <option value="2">2x</option>
  </select>
</div>
```

#### 5. **爆炸深度滑块**
**问题**：只能完全爆炸或按步骤，不能自由控制深度
**方案**：添加一个滑块，允许 0-100% 任意调节爆炸程度

```html
<input type="range" id="explode-depth" min="0" max="100" value="0" />
<label>爆炸深度</label>
```

#### 6. **部件信息卡片**
**问题**：用户想了解某个部件的详细信息时只能看文字
**方案**：
- 悬停/点击部件 → 弹出信息卡片
- 卡片内容：名称、材质、功能、规格
- 3D 视图中的小标签跟随部件

```javascript
const partInfo = {
  '前面板': {
    name: '前面板 (Front Cover)',
    material: 'PC 塑料 (聚碳酸酯)',
    weight: '约 45g',
    function: '保护内部组件，提供佩戴支撑',
    specs: '尺寸：129×70×12mm，白色哑光'
  }
};
```

#### 7. **键盘快捷键**
**问题**：只能用鼠标点击，效率低
**方案**：
- `←` / `→`：上一步/下一步
- `Space`：爆炸/合体
- `R`：重置
- `A`：自动旋转开关
- `F`：聚焦当前步骤部件

## 🔧 技术债务

### 代码优化

#### 17. **模块化重构**
**当前问题**：
- `main.js` 840+ 行，太长
- 没有模块划分
- 难以维护

**方案**：
```
src/
├── main.js          # 入口
├── scene/
│   ├── init.js      # 场景初始化
│   ├── lights.js    # 灯光系统
│   └── materials.js # 材质库
├── model/
│   ├── quest3.js    # Quest 3 模型构建
│   ├── parts.js     # 部件管理
│   └── explode.js   # 爆炸动画
├── ui/
│   ├── controls.js  # UI 控制
│   ├── steps.js     # 步骤管理
│   └── upload.js    # 文件上传
└── utils/
    ├── loader.js    # GLTFLoader
    └── helpers.js   # 工具函数
```

#### 18. **TypeScript 迁移**
**好处**：
- 类型安全
- 更好的 IDE 支持
- 减少运行时错误

#### 19. **性能优化**
**当前问题**：
- 没有 LOD（细节层次）
- 没有实例化渲染
- 阴影质量过高

**方案**：
- 添加 LOD：远距离降低几何体精度
- 合并静态 Mesh
- 动态阴影：只在动画时开启
- 纹理压缩（如果加载真实模型）
