# 自定义模型加载修复说明

## 🐛 问题

**症状**：上传自定义模型（quest3分解.glb）后，虽然显示"成功加载，检测到 2 个独立部件"，但 3D 视图中看不到新模型。

**根本原因**：
1. 默认的简化版 Quest 3 模型（`questGroup`，15 个部件）仍然可见
2. 自定义模型（`customModelGroup`）也被添加到场景中
3. 两个模型重叠，导致看起来像是什么都没发生

## 🔧 修复方案

### 代码变更

#### 1. `loadCustomModel()` - 隐藏默认模型

```javascript
hasCustomModel = true;

// 隐藏默认模型，显示自定义模型
questGroup.visible = false;

URL.revokeObjectURL(url);
```

#### 2. `clearCustomModel()` - 恢复默认模型

```javascript
hasCustomModel = false;

// 恢复默认模型可见性
questGroup.visible = true;
```

## 📊 你的模型情况

### quest3分解.glb

- **部件数量**: 2 个
- **爆炸效果**: ⚠️ **可能不明显**

**原因**：
- 爆炸视图是根据部件之间的距离计算爆炸方向的
- 只有 2 个部件时，如果它们距离很近，爆炸效果会很微弱
- 默认 Quest 3 有 15 个部件，分布在不同方向，爆炸效果明显

**建议**：
1. ✅ **尝试拖动爆炸深度滑块**（如果 UI 上有）
2. ✅ **点击"爆炸视图"按钮**进行完全爆炸
3. ✅ **尝试上传一个部件更多的模型**（最好是 5+ 个独立部件）

## ⚙️ 技术细节

### 问题根源

```javascript
// ❌ 之前的代码
const questGroup = new THREE.Group();
scene.add(questGroup);  // 添加到场景

const customModelGroup = new THREE.Group();
scene.add(customModelGroup);  // 也添加到场景

// 两个 group 都在场景中，导致重叠
```

### 解决方案

```javascript
// 加载自定义模型时
questGroup.visible = false;  // 隐藏默认模型
customModelGroup.visible = true;  // 显示自定义模型

// 清除自定义模型时
questGroup.visible = true;  // 恢复默认模型
customModelGroup.visible = false;  // 隐藏自定义模型
```
