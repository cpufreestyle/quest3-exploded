# 自定义模型爆炸效果调试指南

## 问题：模型太小 + 无法爆炸

### 原因分析

#### 1️⃣ 模型太小

自定义模型的尺寸可能与默认的简化版 Quest 3 不同，导致看起来很小。

**可能的原因：**
- 模型本身的尺寸较小
- 模型缩放比例不同
- 相机距离没有自动调整

#### 2️⃣ 爆炸效果不明显

你的模型 **quest3分解.glb** 只有 **2 个部件**，这是爆炸效果不明显的根本原因。

**为什么 2 个部件的爆炸效果差？**

```
爆炸距离 = 部件距中心距离 × 5

如果两个部件都在中心附近：
- 部件 A 距中心 0.05 → 爆炸距离 0.25
- 部件 B 距中心 0.08 → 爆炸距离 0.40

总爆炸幅度只有 0.65，对比默认模型的 2-3 单位爆炸，几乎看不到效果！
```

默认的简化版 Quest 3 有 **15 个部件**，分布在不同的方向：
- 前面板：前方 0.9 单位
- 主机身：中心（不移动）
- 透镜：左右各 0.8 单位
- 头带：后方 1.5+ 单位
- 等等...

**推荐模型类型（爆炸效果好）：**

1. **机器人/机械臂** 🤖
   - 10-20 个独立部件
   - 四肢、头部、躯干等分开
   - 爆炸距离大

2. **汽车模型** 🚗
   - 引擎、轮子、车门、车顶等
   - 10+ 个部件
   - 爆炸效果好

3. **电子产品拆解** 📱
   - 手机、笔记本内部结构
   - 屏幕、电池、主板等分开
   - 8-15 个部件

4. **建筑模型** 🏢
   - 楼层、房间分开
   - 5-10 层
   - 爆炸效果明显

**在哪里找测试模型？**
- Sketchfab.com（免费 GLB 模型）
- Three.js 官方示例
- CGTrader.com

#### 方案 D: 修改爆炸倍率（代码级）

如果需要更强的爆炸效果，可以修改代码：

**位置**: `main.js` 第 ~409 行

**当前代码**:
```javascript
const explodeDist = Math.max(worldPos.distanceTo(center) * 5, 0.5);
```

**修改为**（增强爆炸）:
```javascript
const explodeDist = Math.max(worldPos.distanceTo(center) * 10, 1.0);
```

**说明**：
- `* 5` → `* 10`：爆炸距离翻倍
- `0.5` → `1.0`：最小爆炸距离增加到 1.0

## 📊 预期行为

### 正常的爆炸流程

```
初始状态（步骤 0）:
  - 所有部件在 homePos
  - customFactor = 0

点击"爆炸"或拖动滑块:
  - customFactor 从 0 → 1
  - 部件从 homePos → explodePos
  - 应该看到部件明显分开

完全爆炸（customFactor = 1）:
  - 所有部件到达 explodePos
  - 距离 = explodeDist
  - 应该看到明显的分离效果
```

### 你的模型可能出现的情况

**情况 1: 部件确实分开了，但因为太小看不到**
- ✅ 解决：缩放视角（滚轮），或者换个角度

**情况 2: 部件确实没动**
- ❌ 检查控制台是否有错误
- ❌ 检查 customFactor 是否为 0
- ❌ 检查部件数量是否为 0

**情况 3: 部件动了但幅度很小**
- ⚠️  正常！2 个部件 + 距离近 → 爆炸幅度小
- 💡 建议上传更多部件的模型
